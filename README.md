# Sobre mis colaboraciones académicas

En mi perfil de GitHub aparecen repositorios que corresponden a **proyectos académicos desarrollados en equipo durante la universidad**.

En varios de estos proyectos:
- El trabajo se realizó de forma **grupal**
- **Un solo miembro del equipo centralizaba los commits**, siguiendo las instrucciones de la materia
- Mi participación fue **técnica y activa**, aunque no siempre reflejada directamente en el historial de commits

A continuación detallo **mi rol y las tecnologías utilizadas** en cada proyecto.

## Redibo – Plataforma web de renta de automóviles

Proyecto académico desarrollado para una materia universitaria.  
Consiste en una **aplicación web para la renta de automóviles**, construida bajo metodología SCRUM.

### redibo-back (Backend)

**Rol y participación**
- Desarrollo y apoyo en la lógica de backend
- Colaboración grupal: los commits fueron subidos por un único integrante según las instrucciones del curso

**Tecnologías utilizadas**
- Express
- TypeScript
- Prisma ORM
- PostgreSQL
- PNPM

---

### redibo-front (Frontend)

**Rol y participación**
- Diseño de **mockups en Figma**
- Implementación de componentes visuales:
  - Barra de buscador
  - Slide de recomendaciones
- Apoyo general al equipo de frontend

**Tecnologías utilizadas**
- Figma
- Next.js
- React
- Tailwind CSS
- NPM

---

### Gestión del proyecto
- Planificación y seguimiento usando **JIRA**
- Metodología **SCRUM**

##  Multi-Agent Shopping

Proyecto académico enfocado en **sistemas multiagente**.

El sistema simula un entorno de supermercado donde:
- Un agente debe recolectar productos
- Optimiza la ruta hacia el cajero
- Tiene como limitante un **presupuesto máximo (vale)**

**Rol y participación**
- Creación y definición de agentes
- Apoyo en la planificación del comportamiento de los agentes
- Colaboración parcial en el frontend
- Los commits fueron centralizados por un compañero

**Tecnologías utilizadas**
- Python
- JavaScript
- Tailwind CSS
- CSS
